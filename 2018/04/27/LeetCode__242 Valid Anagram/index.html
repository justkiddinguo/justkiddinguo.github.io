Java Solution1:

  • 要检查t是否是s的重排列，可以计算两个字符串中每个字母的出现次数，并进行比较。
  • 由于s和t只包含来自a-z的字母，因此大小为26的简单计数表就足够了。
  • 需要创建两个表进行比较吗？ No，因为可以++s中每个字母的计数值并–t中每个字母的计数值，然后检查是否回到零。

​ 或者可以先增加s的计数值，然后减去t的计数值。 如果计数值降到零以下，我们知道t包含一个不在s中的字母，立即返回false。

复杂性分析

  • 时间复杂度：O(n)，因为访问计数表是恒定时间操作。

  • 空间复杂度：O(1)，虽然使用了额外的空间，但空间复杂度为O（1），因为无论n多大，表的大小都保持不变。

Java Solutin3：

将s的字母重新排列为t，如果t是s的重排列，对两个字符串进行排序将产生两个相同的字符串。 如果s和t有不同的长度，则t不可能是s的重排列，可以提前返回。

复杂性分析

  • 时间复杂度：O(nlogn)。 假设n是s的长度，排序成本为O(nlogn)，比较两个字符串成本O(n), 排序时间占优,总体时间复杂度为O(nlogn)。

  • 空间复杂度：O(1)。 空间取决于排序实现，如果使用heapsort，通常会花费O(1)辅助空间。 在Java中，toCharArray（）会生成字符串的副本，因此它需要额外的O（n）空间，但我们忽略了这一点以进行复杂性分析，因为：它是一种语言相关的细节,这取决于功能的设计方式。 例如，函数参数类型可以更改为char []。

Follow Up：

如果输入包含unicode字符怎么办？

Answer：

使用哈希表而不是固定大小的计数表。 如果分配一个大尺寸数组以适应整个unicode字符范围，数目可能会超过100万。 哈希表是一种更通用的解决方案，可以适应任何范围的字符。
